OCCI-WG Conference Call 2009-04-22

Note: Future conference calls to be held at 08h00 PDT / 17h00 CEST

When: 04/22/2009 - 4pm CET Where: concall Attendees: 14 people on the call

Agenda:

  1. Introduction
  2. Use Cases and Requirements
  3. API

Introduction

  • Chairs, attendees and current work were introduced. The weekly concalls will be moved one hour back to 5pm CEST.
  • TM requested two slots for OGF 26 - one for presentation (marketing moreso than technical - with slides) and one for discussion/feedback.
  • We will still stick to the approach/timeline to present a first draft of the API @OGF26 and a first final specification in "static" format @ OGF27. After that ongoing enhancement by way of registries (following consensus on the mailing list) and extension development (e.g. firewall management, ) can be made (and are a goal).
  • Overall the work done has been great. In the last week the charter was approved and the infrastructure for the group setup. A press-release is on its way.

Use Cases and Requirements

  • The consensus was the use cases and requirements are important and need to be gathered. More effort needs to be made to push this work.
  • Please take a look at the use case pages and update it if necessary

API Design

  • No action required beyond developing information in the wiki
  • Taxonomy/Ontology to be defined by end April (already mostly complete)
State Model
  1. We have the essential states that we need now
  2. Add states as we need them via registries
  3. This approach should give sufficient flexibility
  4. Keep state diagram very simple (with comment about error state being universally applicable)
Nouns, Verbs & Attributes
  1. Compute / Network / Storage (with subtypes for e.g. physical/virtual machines and networks, drives and snapshots, etc.)
  2. IP addresses, VLANs, etc. (IDs inside networks is ok for EH - details need to be codified and ratified)
  3. High level agreement / consensus on most issues - little contention

Action Items

Mail about requesting more use cases - better to get input now than later.

Remarks

Deadlines
  • April - Nouns / Verbs / Attributes (Wiki)
  • May 15 - Formats / Wire Protocol (Wiki)
  • May 22 - Internal Feedback
  • May 29 - External Feedback
  • June - Documentation

Open Todos

  1. Look for other issues potentially in scope (Network management API (basic)/Storage management API (basic)/Security functionality (basic))
  2. Work on document drafts (OGF templates etc)
