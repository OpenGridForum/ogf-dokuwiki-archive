Use Cases

Overview

This page provides information to the Cloud community regarding use-case scenarios involving provisioning, monitoring and definition of Cloud Infrastructure (IaaS) services. It does not define any standards or technical recommendations. The scope is limited to the high level functionality required for life-cycle management of virtual machines.

Types of consumers are:

  • Service management platforms building same solution of multiple clouds
  • Technologies for building hybrid clouds
  • Service providers using public clouds
  • Integrators offering advanced management services
  • End users

Use Case Template

This is a template for providing use cases to the OCCI "Open Cloud Computing Interface" Working Group (OCCI-WG). The requirements from these use cases will be used to design the OCCI-API specification.

  • Name: Name of the use case
  • Contact: Contact person and organization
  • Type of Organization: Industry/Academic/Other
  • Description: 3 paragraphs with brief description and figure
  • URL: URL with more information about the use case
  • Functional Requirements: These should represent method calls in the API specification. The following subsections are optional and new subsections can be added. Functional areas could be
    • VM Description: Attributes required in the VM description, such as name, resource requirements (CPU, memory...)...
    • VM Management: States in the VM life-cycle, monitoring information, VM controlling, submission of groups of VMs...
    • Network Management: Creation and mapping of interfaces, assignment of these to public or private networks and assignment of dynamic or static IPs
    • Image Management: Preregister and upload of images.
  • Non-functional Requirements: Although not represented in the API as additional calls, the API should allow implementations to support these functionalities. The following subsections are optional and new subsections can be added. Non-functional areas could be:
    • Security: Authentication
    • Quality of Service: Advance reservation of capacity...
