Implementors

The following companies and projects have made statements about implementing the OCCI API:

Providers

  • Cisco Systems - An internal implementation - more emphasis on the network and compute APIs , would be glad to participate in interop tests et al
  • CloudCentral, CloudCentral will also be implementing OCCI (ref)
  • ElasticHosts, With the latest design, OCCI looks like it'll be an API which ElasticHosts will be enthusiastic to implement (ref)
  • FlexiScale, We made an open pledge some months back that we would willingly adopt a new API standard based on what made most commercial sense, and I certainly think OCCI has come a long way towards this (ref)
  • GoGrid, strong participation by Randy Bias (ref)
  • Sun Microsystems, We are going to run with our CC-licensed REST API until we have some experience operating a cloud-computing service because we think we have a whole lot to learn about where the real pain points are (ref)

Open Source

  • RESERVOIR: Reference implementation on top of OpenNebula and plug-in to access OCCI (ref)
  • SLA@SOI: implementation of OCCI on the infrastructure management layer
