Modeling Users

  • For GLUE 2.0, we do not address the issue of modeling single
users as an independent concept; we only model collection of users; for the presented use case (user-oriented job monitoring), this can be addressed in two possible ways:
    • using the Owner attribute of the ComputingActivity;
    • model users a single-person userDomain and use associations to the Computing Activity (see tne newly added UserDomain.Member attribute)
