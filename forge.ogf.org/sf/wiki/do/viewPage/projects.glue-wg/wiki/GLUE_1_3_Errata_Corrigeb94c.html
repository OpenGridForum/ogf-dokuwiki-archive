Errata Corrige for GLUE 1.3 Spec

How to submit errata corrige

  1. Submit request to the GLUE mailing list
  2. The request will be discussed in a GLUE call or in the mailing list and then added to this page once approved

EC 1:

The attribute AccessControlBase.Rule has multiplicity * instead of 1 in all the tables

EC2:

Policy.Preemption in Computing Element is defined as int32, while in VOView is defined as boolean; According to the current LDAP implementation, both should be int32 and with the same definition

EC3:

cestatus_t in data type UML diagram, page 21; values all have first letter capitalized

EC4:

missing sestatus_t in data type UML diagram, page 21; it should be like cestatus_t; values all have first letter capitalized

EC5:

for attributes which type is integer, if the information provider reports a real value, then this should be rounded up (i.e., the smallest integer greater than or equal to the measured value should be used)

EC6:

There is a typo in the 1.3 schema document (CE and VOView, pages 10 and 11):

"Policy.MaxSlotsPerJobs int32 1 The maximum number of slots which could be allocated to a single job (defined to be 1 for a site accepting only standard jobs). (NN)"

That should be MaxSlotsPerJob (singular) - it's correct in the LDAP schema.
