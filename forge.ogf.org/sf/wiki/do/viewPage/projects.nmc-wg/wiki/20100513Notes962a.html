!Result Codes

  • See Roman's Mail from 4/28
  • pS MDM Result Codes
    • result_codes-mdm-rrdma.txt

  • pS PS Result Codes
    • psps_result_codes.txt
  • Opinion of some - when comparing the old to the proposed, its a total mess. Difficult to map going forward. Somethings can fit into multiple categories.
  • Aaron wants to see us go through the list and classify as broadly as we can first. If things are similar, they live together. If things fit into two categories, label them as such (for a first cut)
  • Be careful with events vs errors, errors are more important.
  • Note Jason's comments on the mailing list about re-use and how it can be bad sometimes.
  • Also need to be sure we need the old things - if its doesn't make sense (or is spurious) we may not need it.
  • Roman muses on what is 'bad' vs 'not acceptable' when it comes to errors, and also what happens when there is overlap. Our goal should be to eliminate any confusion to minimize the overlap.
  • Jeff notes that the early categorization will eliminate overlap and dups.
  • Some talk on a recent suggestion to the mailing list regarding structuring the URLs in such a way that a client can parse it so far and know what it is - then the developer can create own messages. Jason cautions against this - we need to be firm and define things strictly.
  • Actions set to go further with the current codes, and stick them in the new tree.
