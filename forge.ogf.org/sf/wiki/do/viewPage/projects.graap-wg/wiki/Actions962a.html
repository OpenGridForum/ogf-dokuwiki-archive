wiki1780: Actions

Actions, issues, etc.

PENDING & ACTIVE

This is a list of issues, actions, et cetera currently of importance for GRAAP. One reason to have this here is to better keep track ot the overall workload, but also to keep track of developments and decisions even after the case is cold. Time-critical tasks with will find their way into the task manager anyway (the respective tasks will be linked here).

  • (A0001) Description of WS-Agreement interopability scenarios (ACTIVE; Oliver, Wolfgang, Dominic, Omer; before OGF 21).
    • Structure & outline have been discussed during OGF 20.
    • Currently this is done using the following Google document http://docs.google.com/Doc?id=dd2wwjk8_0d7b58b (you need a Google account for accessing it).
  • (A0004) Define application domains for SLAs, use cases (PENDING; ?; before OGF 21)
    • A number of use cases already exists. They have to be collected.
    • Target: Informational document.
    • Extract common terms from the use cases (see also A0005).
    • Could result in micro-specs (see e.g. Asit's ideas on Key Performance Indicators from GGF 14).
  • (A0005) Common set of service description terms (incl. GSA_RG terms, OGSA terms, etc.) (PENDING; ?; ?)
    • Some members of GRAAP attended the BoF on Service Level Terms for OGSA-ByteIO and OGSA-BES, and expressed out interest in a common set of terms, but besides that, according to Wolfgang, nothing happened so far.
    • During the May 23, 2007 telecon it has been decided to start with a few domains and check the common terms.
    • Since Andrew Grimshaw seems to be driving this from the OGSA side, Wolfgang will contact him to further discuss this issue.
    • Contact to GSA-RG wrt to scheduling attributes can easily be done (Philipp).
  • (A0006) Micro-spec for re-negotiation (ACTIVE; Toshi; before OGF 21) Toshi: I am trying to list things to ponder in the Re-Negotiation Wishlists
    • First input has been received on May 23, 2007.
  • (A0007) Contact Globus Alliance concerning roadmap towards WSRF/WS-Adressing spec. compliance (PENDING; Dominic; asap)
  • (A0008) Input to question on how to minimize cost of negotiation (PENDING; Omer; after OGF 21)
  • (A0010) Telecon with projects like BEinGrid, Trustcom (PENDING; Wolfgang; ?)
    • Gather list of contact points first.
    • Maybe a more business-oriented WS at OGF 21 may be an opportunity to get input to a next version of WS-Agreement.

DONE

  • (A0002) List of existing and planned implementations (DONE_)
    • We started with a preliminary list during OGF19 and should contact these projects/people now, given that WS-Agreement is now a proposed recommendation.
  • (A0003) Check status of the CREMONA implementation (DONE)
    • Asit gave an answer to the question in one of the last telecons. The answer was, that the CREMONA project has been completed a while ago and for the time being there are no concrete plans to resume this. Nevertheless this could happen some day.
  • (A0009) Contact Paul Strong wrt to OGSA terms (DONE)
    • Comment from Toshi: "I talked with Paul a little bit on OGF20. I think there is some confusion. It seems Paul had been talking about 'Term' Deployment for OGSA Glossary. He was willing to give us input on what he would see as necessary for deployment needs."
