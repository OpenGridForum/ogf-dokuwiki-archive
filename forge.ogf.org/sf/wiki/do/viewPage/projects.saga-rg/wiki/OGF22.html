OGF 22 Activities

Monday, February 25, 9:30 am - 11:00 am, Crispus Attucks
SAGA, Java language binding
(SAGA-CORE-WG) Group Discussion Responsible: Thilo
Material:

  • draft java binding (GridForge)

The status of the Java language binding is presented, based on the results from the discussion at OGF21, and as backed by the ongoing reference implementation by VU University Amsterdam. Feedback for improvements is solicited in order to advance the work such that it can be put into a recommendation-track document.

Agenda:

  1. Opening, IPR issues, agenda bashing
  2. Presentation of the state of the language binding, with respect to Java-specific issues
  3. Discussion, action items, planning for a recommendation-track document.

Monday, February 25, 11:15 am - 12:45 pm, Crispus Attucks
SAGA, Extension packages
(SAGA-RG) Group Discussion
Responsible: Andre
Material:

  • draft: RPC data handles
  • draft: RPC v.2 with data handles
  • draft: Service Discover extension
  • draft: MessageBus extension
  • draft: CPR extension
  • draft: Advert extension

Beyond the SAGA-CORE, with its "core" functionality packages, further packages are deemed useful, and work is ongoing. In this session , we present and discuss such packages, aiming to solidify work towards extension recommendation-track documents.

Agenda:

  1. Opening, IPR issues, agenda bashing
  2. The service discovery package
  3. The message bus package
  4. Which other packages do we need?

Monday, February 25, 3:30 pm - 5:00 pm, Cambridge
SAGA + DRMAA
(SAGA-RG + DRMAA-WG) Group Discussion
Responsible: Andre
Material:

  • draft: DRMAA IDL Rendering
  • draft: DRMAA extension

This session will host a discussion between SAGA and DRMAA, to define future collaboration modes and milestones.

Agenda:

  1. short (!) outline of SAGA extension approach
  2. discussion of SAGA/DRMAA relationship, and of the potential/need for a DRMAA extensions to SAGA.

Monday, February 25, 5:30 pm - 7:30 pm, Ballroom A/B/C
Reception and Spec Adoption Demonstrations
Responsible: Thilo

The Simple API for Grid Applications (SAGA)
SAGA is a high level, simple to use, application-oriented API for grid application development. The SAGA specification describes simple API's for commonly needed operations for file transfer and access, replica management, stream based interprocess communication, job submission and monitoring, and client side RPC calls. This demonstration will show a functioning C++ implementation of many of the core packages and a usable SAGA based Grid shell.

Agenda:

  1. Demo: (Andre Merzky)
  2. Poster: (Thilo Kielmann)

Tuesday, February 26, 1:45 pm - 3:15 pm, Molly Pitcher
SAGA: The Simple API for Grid Applications
(Spec Adption Forum)
Responsible: Shantenu

The Simple API for Grid Applications (SAGA) has been designed to provide a simple, stable, and middleware-independent API for grid applications. We start presenting the API itself (the SAGA CORE API), along with ongoing work on extension packages. Then, projects adopting SAGA present their efforts and rationale for using SAGA. (e.g., OMII-UK, DEISA, NAREGI; speaker invitations pending)
