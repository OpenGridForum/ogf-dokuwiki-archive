Planned Demonstrations

GINSC2008-001: Scientific bloodflow simulation using interoperability between NGS, TeraGrid, and DEISA (In Preparation)

This demonstration shows a bloodflow simulation (i.e. HemeLB) that requiring HPC resources in more than one Grid. The scientists, most notably Peter Coveney et al., will use their application hosting environment to access NGS via GridSAM and DEISA via UNICORE 6. The demonstration will show the OGSA-BES standard in a real scientific use case.

Contact: Morris Riedel, Stefan Zasada, Steven Manos, Johannes Reetz, Michael Rambadt
Category: Production
Status: in preparation

More information

GINSC2008-002: Scientific biomedical application workflow that is using interoperability between EGEE and DEISA (In Preparation)

This demonstration shows a complete scientific workflow using docking applications (i.e. Autodock, FlexX) and MD simulations (i.e. AMBER) on the Grids EGEE and DEISA. Therefore, this demonstration involves technologies from gLite and UNICORE 6. The demonstration will show the OGSA-BES standard in a real scientific use case.

Contact: Morris Riedel, Vinod Kasam
Category: Production
Status: in preparation

More information

GINSC2008-003: SRM and SRB interoperation

The main aim of this demo is to show that we can still interoperate SRM and SRB by transferring files between two grids: OpenScienceGrid running SRMs, and SRB run by Rutherford Appleton Laboratory. This demo focuses on using core Globus tools to use the transfer using the GridFTP built for SRB by ANL. Contact: Jens Jensen, Alex Sim
Category: Production
Status: ready

More information

GINSC2008-004: SAGA for Application Level Grid interoperation

The CCT/LSU boost will give a quick introduction to SAGA, demonstrations of several distributed applications, and a talk covering several other applications. SAGA applications will span multiple Grid and Cloud infrastructures (LONI, TeraGrid, EC2, Nimbus, Eucalyptus). Contact: Shantenu Jha
Category: Demo + Talk
Status: ready

More information

GINSC2008-005: UNICORE 6 Commandline Client (UCC) vs. A-REX (KnowARC/NorduGrid OGSA-BES service, ARC1)

The demo shows how UCC client can use A-REX BES standard-compliant interface, thus that the interoperability between Unicore client and ARC server works. The main demo items are as follows:

-job submission to A-REX
-job status query
-job termination
-A-REX's BES-Factory attributes query
-job listing from A-REX

Contact: Peter Stefan
Category: Demo + Talk
Status: ready

More information

GINSC2008-006: ARC1 clients vs. CREAM2 service (gLite)

This demo illustrates the job submission/termination/status query between ARC1 client and CREAM2 computing element. Other interesting topic is xRSL translation to other job description languages: JSDL, JDL.

Contact: Peter Stefan
Category: Demo + Talk
Status: ready

More information

GINSC2008-007: Interoperability between production UNICORE 5 and production gLite

This demo illustrates the exchange of jobs between production UNICORE 5 and production gLite using proprietary interfaces.

Contact: Daniel Mallmann (d.mallmann@fz-juelich.de), Morris Riedel
Category: Demo + Talk
Status: ready
