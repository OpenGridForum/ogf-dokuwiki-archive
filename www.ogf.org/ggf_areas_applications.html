Area Scope:
The Applications Area explores grid application development issues and programming interfaces required by grid applications.

Applications Groups:

Grid Remote Procedure Call (GridRPC-WG)
The GridRPC Working Group is chartered to produce a Recommendation GGF document for a Grid Remote Procedure Call Model and API.

Grid Information Retrieval (GIR-WG)
The GIR WG will establish a specific set of requirements, architecture, and detailed specifications for a particular Information Retrieval (IR) system on the OGSA Grid. GIR will provide document collection management, indexing/searching, and query processing services to OGSA grid users and applications.

Distributed Resource Management Application API (DRMAA-WG)
This group will develop an API specification for the submission and control of jobs to one or more Distributed Resource Management (DRM) systems. The scope of this specification is all the high level functionality which is necessary for an application to consign a job to a DRM system including common operations on jobs like termination or suspension.

Simple API for Grid Applications (SAGA-RG)
The group will lower the barrier for scientific application developers to make use of the grid by providing a small, consistent API for the operations of interest, the Simple API for Grid Applications (SAGA).

Grid Checkpoint Recovery (GridCPR-WG)
The Grid Checkpoint Recovery Working Group will define a user-level API and associated layer of services that will permit checkpointed jobs to be recovered and continued on the same or on remote Grid resources.
