Just to throw a format out there that seems to have been left off the list
and yet was designed from the ground up as an efficient data serialization
format for decentralized grid/cloud computing, what about Protocol Buffers?
 I realize that using ProtofBuf in a majority of use cases being discussed
on this list would be extreme overkill.  But by its very nature, cloud
computing tends to encourage the hucking of large amounts of data from one
node and from one cloud to the next.  In cases such as this where Interop
between disparate platforms, systems, languages, etc. come par for the
course, and therefore cleanly fit into the OCCI charter, shouldn't
considerable thought be put into using the format best suited for the task
at hand?
