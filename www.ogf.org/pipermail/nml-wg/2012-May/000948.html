Just a quick overview of the notes:

Aaron, Freek, Chin and Jeroen

- Label proposal from Roman on the list describes the outcome of the discussion last week.
Freek notes that we should only allow one label per port. This makes it possible to described configured networks very well.
It's not so suitable for describing available labels, because then you would have to describe each available label individually. We will find some solution to this later.
Freek will make an RDF translation for the proposal and put it on the issue tracker.

Freek posts two followups and concludes with a more generic solution for the ordered lists.
Jeroen will update the issue on the tracker.

- Looking for authors:
* Freek will add a list of relations to the document
* Jeroen will post a list of issues available for inclusion to the list, along with subversion instructions for the document.
