bug #160

extensible structs need to have a void * pointer to point to extensions

Added by Daniel Gruber about 8 years ago. Updated about 7 years ago.

Status:closed Start date:08/26/2014
Priority:High Due date:
Assignee:Daniel Gruber % Done:

0%
Category:-
Target version:-

Description

When having an extensible struct like the job template
there needs to be a reference to the extension itself.
This is best done by a void * pointer which should only
be accessed by the DRMAA2 implementation. Without any
reference there is no mapping possible.

Example from Univa Grid Engine drmaa2.h:

307 drmaa2_dict stageOutFiles;
308 drmaa2_dict resourceLimits;
309 drmaa2_string accountingId;
310 void* implementationSpecific;
311 } drmaa2_jtemplate_s;

This implementation specific pointer is then casted internally
to a reference of a struct which contains all Univa Grid Engine
specific extensions.

All extensible data structures have such a requirement.
