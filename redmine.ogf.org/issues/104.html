bug #104

need to clearify that returned lists are initialized with callback functions

Added by Daniel Gruber about 9 years ago. Updated about 7 years ago.

Status:closed Start date:08/23/2013
Priority:Normal Due date:
Assignee:- % Done:

0%
Category:-
Target version:-

Description

We have functions that return lists (like job lists, string lists and
so on). We need to clarify in the spec that the lists need be initialized
with proper callbacks in the vendor implementation so that not memory
leaks remain. The user just have to call the list free function in order
to reap the whole list without memory leaks.

Updated by Peter Tröger about 9 years ago

I don't get that. It sounds like an implementation guide ?!? The free semantics are crystal clear, so all developers must derive the according rules for their memory management.

Updated by Peter Tröger about 8 years ago

  • Status changed from proposed to final review

The OGF-42 meeting decided upon the following text clarifications:

  • List and dictionary items must be deleted by the producer of them, which is either the application or the implementation itself.
  • If the application is expected to do that, then the callback can be used. This should become part of the C library documentation.
