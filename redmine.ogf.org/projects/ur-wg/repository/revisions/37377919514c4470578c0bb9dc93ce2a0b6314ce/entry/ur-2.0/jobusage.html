\subsection{GlobalJobId}


    		
  

  

  

    
      14
    
    
      

    		
  

  

  

    
      15
    
    
      
This property describes the global identity of the job. The property should identify the job globally, such that clashes do not happen accidentally. This could for example be a combination of time stamp, local job id and host name.


    		
  

  

  

    
      16
    
    
      
\begin{itemize}


    		
  

  

  

    
      17
    
    
      
\item The \emph{GlobalJobId} field type MUST be a string.


    		
  

  

  

    
      18
    
    
      
\end{itemize}

      32

\subsection{Charge}


    		
  

  

  

    
      36
    
    
      

    		
  

  

  

    
      37
    
    
      
This is property may be used for economic accounting purpose. It is the charge applied to the user resource usage.


    		
  

  

  

    
      38
    
    
      
\begin{itemize}


    		
  

  

  

    
      39
    
    
      
\item When specifying \emph{Charge}, the following two attributes MUST be specified:


    		
  

  

  

    
      40
    
    
      
\begin{itemize}


    		
  

  

  

    
      41
    
    
      
\item \emph{unit} – defines the currency used to report the charge as defined in ISO 4217\footnote{http://www.iso.org/iso/currency\_codes}.


    		
  

  

  

    
      42
    
    
      
\item The \emph{unit} attribute type MUST be an ISO 4217 alphabetic code.


    		
  

  

  

    
      43
    
    
      
\item \emph{formula} – The cost computation formula adopted to compute the applied charge.


    		
  

  

  

    
      44
    
    
      
\item The \emph{formula} attribute type MUST be a string.


    		
  

  

  

    
      45
    
    
      
\end{itemize}


    		
  

  

  

    
      46
    
    
      
\item The \emph{Charge} field type MUST be a float.


    		
  

  

  

    
      47
    
    
      
\end{itemize}

\subsection{Middleware}


    		
  

  

  

    
      68
    
    
      

    		
  

  

  

    
      69
    
    
      
The purpose of this element is to mark whether the job was submitted locally or through a grid middleware. At least the values ``local'' and ``grid'' MUST be supported. The description attribute SHOULD be used to give additional information on the used middleware.


    		
  

  

  

    
      70
    
    
      
\begin{itemize}


    		
  

  

  

    
      71
    
    
      
\item \emph{Middleware} MUST be present.


    		
  

  

  

    
      72
    
    
      
\item The values ``local'' and ``grid'' MUST be supported.


    		
  

  

  

    
      73
    
    
      
\item The \emph{Middleware} field type MUST be a string.


    		
  

  

  

    
      74
    
    
      
\item The \emph{description} attribute SHOULD be used.


    		
  

  

  

    
      75
    
    
      
\item The \emph{description} attribute type MUST be a string.


    		
  

  

  

    
      76
    
    
      
\end{itemize}

\subsection{TimeInstant}


    		
  

  

  

    
      88
    
    
      

    		
  

  

  

    
      89
    
    
      
This element may be used to insert in the records any time instant related to the user payload. Three optional values for the type attribute are defined as they are of common usage by batch systems. The semantic is derived from Torque.


    		
  

  

  

    
      90
    
    
      
\begin{itemize}


    		
  

  

  

    
      91
    
    
      
\item \emph{TimeInstant} SHOULD be present in the record.


    		
  

  

  

    
      92
    
    
      
\item \emph{TimeInstant} MAY be present multiple times.


    		
  

  

  

    
      93
    
    
      
\item The \emph{TimeInstant} field type MUST be an ISO 8601:2004 timestamp.


    		
  

  

  

    
      94
    
    
      
\item The following three types are defined and SHOULD be reported:


    		
  

  

  

    
      95
    
    
      
\begin{itemize}


    		
  

  

  

    
      96
    
    
      
\item \emph{Ctime} - Time job was created


    		
  

  

  

    
      97
    
    
      
\item \emph{Qtime} - Time job was queued


    		
  

  

  

    
      98
    
    
      
\item \emph{Etime} - Time job became eligible to run


    		
  

  

  

    
      99
    
    
      
\end{itemize}


    		
  

  

  

    
      100
    
    
      
\item The \emph{type} attribute type MUST be a string.


    		
  

  

  

    
      101
    
    
      
\end{itemize}


    		
  

  

  

    
      102
    
    
      

    		
  

  

  

    
      103
    
    
      
\begin{XMLexample}


    		
  

  

  

    
      104
    
    
      
<ur:TimeInstant ur:type="Etime">2013-05-31T10:59:42</ur:TimeInstant>


    		
  

  

  

    
      105
    
    
      
\end{XMLexample}

\subsection{ServiceLevel}


    		
  

  

  

    
      108
    
    
      

    		
  

  

  

    
      109
    
    
      
This element is used to insert computing benchmarks and normalization factors.


    		
  

  

  

    
      110
    
    
      
At least one normalization factor MUST be present in the record.


    		
  

  

  

    
      111
    
    
      
\begin{itemize}


    		
  

  

  

    
      112
    
    
      
\item \emph{ServiceLevel} MUST be present in the record.


    		
  

  

  

    
      113
    
    
      
\item \emph{ServiceLevel} MAY be present multiple times.


    		
  

  

  

    
      114
    
    
      
\item The \emph{ServiceLevel} field type MUST be of type float.


    		
  

  

  

    
      115
    
    
      
\item At least the following types should be supported:


    		
  

  

  

    
      116
    
    
      
\begin{itemize}


    		
  

  

  

    
      117
    
    
      
\item \emph{Si2k} – SpecInt2000


    		
  

  

  

    
      118
    
    
      
\item \emph{Sf2k} – SpecFloat2000


    		
  

  

  

    
      119
    
    
      
\item \emph{HEPSPEC} – HEPSpec


    		
  

  

  

    
      120
    
    
      
\end{itemize}


    		
  

  

  

    
      121
    
    
      
\item The \emph{type} attribute MUST be present in the element.


    		
  

  

  

    
      122
    
    
      
\item The \emph{type} attribute type MUST be a string.


    		
  

  

  

    
      123
    
    
      
\end{itemize}


    		
  

  

  

    
      124
    
    
      

    		
  

  

  

    
      125

\subsection{Status}


    		
  

  

  

    
      130
    
    
      

    		
  

  

  

    
      131
    
    
      
This property will represent the completion status of the job. For example, this may


    		
  

  

  

    
      132
    
    
      
represent the exit status of an interactive running process or the exit status from the batch


    		
  

  

  

    
      133
    
    
      
queuing system’s accounting record. The semantic meaning of status is site dependent.


    		
  

  

  

    
      134
    
    
      
\begin{itemize}


    		
  

  

  

    
      135
    
    
      
\item The value of \emph{Status} MUST be of type String.


    		
  

  

  

    
      136
    
    
      
\item The \emph{Status} property MUST exist in the record.


    		
  

  

  

    
      137
    
    
      
\item The \emph{Status} property MUST support the following values:


    		
  

  

  

    
      138
    
    
      
\begin{itemize}


    		
  

  

  

    
      139
    
    
      
\item \emph{aborted} – A policy or human intervention caused the job to cease execution.


    		
  

  

  

    
      140
    
    
      
\item \emph{completed} – The execution completed.


    		
  

  

  

    
      141
    
    
      
\item \emph{failed} – Execution halted without external intervention.


    		
  

  

  

    
      142
    
    
      
\item \emph{held} – Execution is held at the time this usage record was generated.


    		
  

  

  

    
      143
    
    
      
\item \emph{queued} – Execution was queued at the time this usage record was generated.


    		
  

  

  

    
      144
    
    
      
\item \emph{started} – Execution started at the time this usage record was generated.


    		
  

  

  

    
      145
    
    
      
\item \emph{suspended} – Execution was suspended at the time this usage record was generated.


    		
  

  

  

    
      146
    
    
      
\end{itemize}


    		
  

  

  

    
      147
    
    
      
\item The \emph{Status} property MAY support other values, as agreed upon within the implementation context.


    		
  

  

  

    
      148
    
    
      
\end{itemize}


    		
  

  

  

    
      149
