        <t>At the moment of writing, a revision of RFC2141 is proposed, which may allow a different set of allowed characters (URN-char) in the NSS than the current URN chars (adding "&amp;" and/or "~", whilst removing the reserved characters "%", "/", "?" and "#"). The intention of this document is to only restrict the syntax of the SNID, and have Grid Forum Documents specify the syntax of the SUBNAMESPACE-SPECIFIC-STRING. If RFC 2141 is to be updated, this document may be revised as well.</t>

            A Uniform Resource Identifier (URI) is a compact sequence of characters


  		





  
  442
  
    
            that identifies an abstract or physical resource.  This specification


  		





  
  443
  
    
            defines the generic URI syntax and a process for resolving URI references


  		





  
  444
  
    
            that might be in relative form, along with guidelines and security


  		





  
  445
  
    
            considerations for the use of URIs on the Internet.


  		





  
  446
  
    
            The URI syntax defines a grammar that is a superset of all valid URIs,


  		





  
  447
  
    
            allowing an implementation to parse the common components of a URI


  		





  
  448
  
    
            reference without knowing the scheme-specific requirements of every


  		





  
  449
  
    
            possible identifier.  This specification does not define a generative


  		





  
  450
  
    
            grammar for URIs; that task is performed by the individual


  		





  
  451
  
    
            specifications of each URI scheme.
