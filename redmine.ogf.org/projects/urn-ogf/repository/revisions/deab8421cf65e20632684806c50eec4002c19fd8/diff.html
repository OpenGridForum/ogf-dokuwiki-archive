href="#rfc.section.1">1.</a>?Introduction</h1><p id="rfc.section.1.p.1">The Open Grid Forum (OGF) is a standardisation development organisation in the field of distributed computing. The OGF produces documents such as working drafts, specifications, and schemata. For more information, see <a href="http://www.ogf.org/">http://www.ogf.org/</a></p><p id="rfc.section.1.p.2">Working groups in the OGF community have expressed the need for global, distributed, persistent identifiers in working drafts and standards. Motivated by this need, the OGF would like to assign URNs to some resources in order to retain unique, permanent, location-independent names for them.</p><p id="rfc.section.1.p.3">This namespace specification is for a formal namespace.</p><h2 id="rfc.section.1.1"><a href="#rfc.section.1.1">1.1</a>?Requirements Language</h2><p id="rfc.section.1.1.p.1">The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in <a href="#RFC2119"><cite title="Key words for use in RFCs to Indicate Requirement Levels">[RFC2119]</cite></a>.</p><h1 id="rfc.section.2"><a href="#rfc.section.2">2.</a>?URN Specification for "ogf" NID</h1><h2 id="rfc.section.2.1"><a href="#rfc.section.2.1">2.1</a>?Namespace ID</h2><p id="rfc.section.2.1.p.1">"ogf" requested.</p><h2 id="rfc.section.2.2"><a href="#rfc.section.2.2">2.2</a>?Registration Information</h2><p id="rfc.section.2.2.p.1">Registration Version Number: 1<br> Registration Date: yyyy-mm-dd <span class="comment" id="RFC-Editor">[<a href="#RFC-Editor" class="smpl">RFC-Editor</a>: Please replace with the date of approval of this document for publication as an RFC]</span> </p><h2 id="rfc.section.2.3"><a href="#rfc.section.2.3">2.3</a>?Declared registrant of the namespace</h2><p id="rfc.section.2.3.p.1">Technical Director<br> Open Grid Forum<br> P.O. Box 2326<br> Joliet, Illinois 60434<br> USA<br> http://www.ogf.org/</p><p id="rfc.section.2.3.p.2">Current contact person: </p><dl class="empty"><dd>Joel Replogle<br> Email: replogle@ogf.org <span class="comment" id="Joel Replogle">[<a href="#Joel%20Replogle" class="smpl">Joel Replogle</a>: Check if a generic mail address exists]</span> </dd></dl><h2 id="rfc.section.2.4"><a href="#rfc.section.2.4">2.4</a>?Declaration of syntactic structure</h2><p id="rfc.section.2.4.p.1">The formal syntax definitions below are given in <a href="#RFC5234">ABNF</a> <cite title="Augmented BNF for Syntax Specifications: ABNF">[RFC5234]</cite>.</p><p id="rfc.section.2.4.p.2">The NSS in the urn:ogf names hierarchy begins with a topic identifier, followed by a delimiter and a topic-dependent string</p><div id="rfc.figure.u.1"></div><pre class="inline">


  		





  
  293
  
    
   OGF-URN  =  "urn:ogf:" TOPIC-ID ":" TOPIC-SPECIFIC-STRING</pre><p id="rfc.section.2.4.p.4">where TOPIC-ID is a unique topic identifier for the specification, and TOPIC-SPECIFIC-STRING is a unique identifier within the topic identifier scope.</p><p id="rfc.section.2.4.p.5">TOPIC-ID has the same syntax as a &lt;NID&gt; as defined in <a href="#RFC2141"><cite title="URN Syntax">[RFC2141]</cite></a>:</p><div id="rfc.figure.u.2"></div><pre class="inline">


  		





  
  294
  
    
   TOPIC-ID  =  ( ALPHA / DIGIT )  *31( ALPHA / DIGIT / "-" )</pre><p id="rfc.section.2.4.p.7">ALPHA and DIGIT are defined in <a href="http://tools.ietf.org/html/rfc5234#appendix-B">Appendix B</a> of <a href="#RFC5234"><cite title="Augmented BNF for Syntax Specifications: ABNF">[RFC5234]</cite></a>.</p><p id="rfc.section.2.4.p.8">The Technical Director at OGF (or it successor) assigns topic identifiers <a href="#URN-OGF"><cite title="URN:OGF Hierarchy Registry and Documentation">[URN-OGF]</cite></a>.</p><p id="rfc.section.2.4.p.9">The syntax of TOPIC-SPECIFIC-STRING is dependent on the TOPIC-ID, and SHOULD be defined by a <a href="#GFD-SERIES">Grid Forum Document</a> <cite title="OGF Document Series">[GFD-SERIES]</cite>. This document does not pose any additional restrictions to the TOPIC-SPECIFIC-STRING than the NSS syntax as defined by <a href="#RFC2141"><cite title="URN Syntax">[RFC2141]</cite></a> or its successor:</p><div id="rfc.figure.u.3"></div><pre class="inline">


  		





  
  295
  
    
   TOPIC-SPECIFIC-STRING  =  1*&lt;URN chars&gt;


  		





  
  296
  
    
</pre><p id="rfc.section.2.4.p.11">URN-char is defined in <a href="http://tools.ietf.org/html/rfc2141#section-2.2">Section 2.2</a> of <a href="#RFC2141"><cite title="URN Syntax">[RFC2141]</cite></a>.</p><h2 id="rfc.section.2.5"><a href="#rfc.section.2.5">2.5</a>?Relevant ancillary documentation</h2><p id="rfc.section.2.5.p.1">The Technical Director at OGF (or it successor) will keep a list of assigned topic identifiers and associated documentation at <a href="http://www.ogf.org/urn/">http://www.ogf.org/urn/</a>.</p><p id="rfc.section.2.5.p.2">This website also contains information on the procedures how to register an topic identifier.</p><h2 id="rfc.section.2.6"><a href="#rfc.section.2.6">2.6</a>?Identifier uniqueness considerations</h2><p id="rfc.section.2.6.p.1">Identifier uniqueness will be enforced by the Technical Director of the Open Grid Forum.</p><h2 id="rfc.section.2.7"><a href="#rfc.section.2.7">2.7</a>?Identifier persistence considerations</h2><p id="rfc.section.2.7.p.1">The Technical Director will only assign topic identifiers for persistent resources <a href="#URN-OGF"><cite title="URN:OGF Hierarchy Registry and Documentation">[URN-OGF]</cite></a>.</p><h2 id="rfc.section.2.8"><a href="#rfc.section.2.8">2.8</a>?Process of identifier assignment</h2><p id="rfc.section.2.8.p.1">Assignment of topic identifiers is limited to the OGF and those authorities that are specifically designated by the OGF Technical Director. OGF may assign portions of its namespace (specifically, those under designated topic identifiers) for assignment by third parties.</p><p id="rfc.section.2.8.p.2">The details of this process will be specified at the OGF website <a href="#URN-OGF"><cite title="URN:OGF Hierarchy Registry and Documentation">[URN-OGF]</cite></a>.</p><p id="rfc.section.2.8.p.3">The syntax and semantics of each TOPIC-ID SHOULD be defined by a <a href="#GFD-SERIES">Grid Forum Document</a> <cite title="OGF Document Series">[GFD-SERIES]</cite> before it is assigned.</p><h2 id="rfc.section.2.9"><a href="#rfc.section.2.9">2.9</a>?Process of identifier resolution</h2><p id="rfc.section.2.9.p.1">OGF will maintain an index of all topic identifiers on its Web site, <a href="http://www.ogf.org/urn/">http://www.ogf.org/urn/</a><span class="comment" id="Joel2">[<a href="#Joel2" class="smpl">Joel2</a>: URL must be created]</span></p><p id="rfc.section.2.9.p.2">Resolution mechanism of TOPIC-SPECIFIC-STRING within each TOPIC-ID SHOULD be defined before assignment of a topic identifier.</p><h2 id="rfc.section.2.10"><a href="#rfc.section.2.10">2.10</a>?Rules for Lexical Equivalence</h2><p id="rfc.section.2.10.p.1">The TOPIC-ID part of URNs in the OGF hierarchy is case insensitive. Thus, the TOPIC-ID MUST be case normalised before comparison.</p><p id="rfc.section.2.10.p.2">The rules for lexical equivalence of the TOPIC-SPECIFIC-STRING part of URNs in the OGF hierarchy is specific for each TOPIC-ID and MUST be defined when a TOPIC-ID is assigned by the OGF Technical Director. These definitions MUST included information about case sensitivity, and in case %-escaped octets, MUST define the exact normalisation to use (interpret as octet, UTF-8, type of Unicode normalisation factor, etc.)</p><h2 id="rfc.section.2.11"><a href="#rfc.section.2.11">2.11</a>?Conformance with URN Syntax</h2><p id="rfc.section.2.11.p.1">Implementors should be aware that a revision of RFC2141 is proposed, which may allow a different set of allowed characters (URN-char) in the NSS than the current URN chars (adding "&amp;" and/or "~", whilst removing the reserved characters "%", "/", "?" and "#"). The intention of this document is to only restrict the syntax of the TOPIC-ID, and have Grid Forum Documents specify the syntax of the TOPIC-SPECIFIC-STRING. If RFC 2141 is to be updated, this document may be revised as well.</p><p id="rfc.section.2.11.p.2">In other words, the current definition:</p><div id="rfc.figure.u.4"></div><pre class="inline">


  		





  
  297
  
    
   TOPIC-SPECIFIC-STRING  =  1*&lt;URN chars&gt;


  		





  
  298
  
    
</pre><p id="rfc.section.2.11.p.4">is likely to change in the future to:</p><div id="rfc.figure.u.5"></div><pre class="inline">


  		





  
  299
  
    
   TOPIC-SPECIFIC-STRING  =  1*&lt;URN-char&gt;


  		





  
  300
  
    
</pre><h2 id="rfc.section.2.12"><a href="#rfc.section.2.12">2.12</a>?Validation mechanism</h2><p id="rfc.section.2.12.p.1">The validation mechanism of URNs in the OGF hierarchy is specific for each TOPIC-ID and MUST be defined when a TOPIC-ID is assigned by the OGF Technical Director.</p><p id="rfc.section.2.12.p.2">URNs in the OGF hierarchy without an assigned TOPIC-ID are considered to be invalid.</p><h2 id="rfc.section.2.13"><a href="#rfc.section.2.13">2.13</a>?Scope</h2><p id="rfc.section.2.13.p.1">Global URNs, relevant for the distributed computing community in general, and the Open Grid Forum in particular.</p><h1 id="rfc.section.3"><a href="#rfc.section.3">3.</a>?Examples (Informative)</h1><p id="rfc.section.3.p.1">The following examples are not guaranteed to be real or even syntactically correct. the urn:ogf namespace has been detected, and a formal registration will allow the Open Grid Forum to regulate its usage and enforce technical review of current practices.</p><h1 id="rfc.section.6"><a 2.4.  Declaration of syntactic structure


  		





  
  173
  
    

  		





  
  174
  
    
   The formal syntax definitions below are given in ABNF [RFC5234].


  		





  
  175
  
    

  		





  
  176
  
    
   The NSS in the urn:ogf names hierarchy begins with a topic


  		





  
  177
  
    
   identifier, followed by a delimiter and a topic-dependent string


  		





  
  178
  
    

  		





  
  179
  
    
      OGF-URN  =  "urn:ogf:" TOPIC-ID ":" TOPIC-SPECIFIC-STRING


  		





  
  180
  
    

  		





  
  181
  
    
   where TOPIC-ID is a unique topic identifier for the specification,


  		





  
  182
  
    
   and TOPIC-SPECIFIC-STRING is a unique identifier within the topic


  		





  
  183
  
    
   identifier scope.


  		





  
  184
  
    

  		





  
  185
  
    
   TOPIC-ID has the same syntax as a <NID> as defined in [RFC2141]:


  		





  
  186
  
    

  		





  
  187
  
    
      TOPIC-ID  =  ( ALPHA / DIGIT )  *31( ALPHA / DIGIT / "-" )


  		





  
  188
  
    

  		





  
  189
  
    
   ALPHA and DIGIT are defined in Appendix B of [RFC5234].


  		





  
  190
  
    

  		





  
  191
  
    
   The Technical Director at OGF (or it successor) assigns topic


  		





  
  192
  
    
   identifiers [URN-OGF].


  		





  
  193
  
    

  		





  
  194
  
    
   The syntax of TOPIC-SPECIFIC-STRING is dependent on the TOPIC-ID, and


  		





  
  195
  
    
   SHOULD be defined by a Grid Forum Document [GFD-SERIES].  This


  		





  
  196
  
    
   document does not pose any additional restrictions to the TOPIC-


  		





  
  197
  
    
   SPECIFIC-STRING than the NSS syntax as defined by [RFC2141] or its


  		





  
  198
  
    
   successor:


  		





  
  199
  
    

  		





  
  200
  
    
      TOPIC-SPECIFIC-STRING  =  1*<URN chars>


  		





  
  201
  
    

  		





  
  202
  
    
   URN-char is defined in Section 2.2 of [RFC2141].

2.10.  Rules for Lexical Equivalence


  		





  
  256
  
    

  		





  
  257
  
    
   The TOPIC-ID part of URNs in the OGF hierarchy is case insensitive.


  		





  
  258
  
    
   Thus, the TOPIC-ID MUST be case normalised before comparison.


  		





  
  259
  
    

  		





  
  260
  
    
   The rules for lexical equivalence of the TOPIC-SPECIFIC-STRING part


  		





  
  261
  
    
   of URNs in the OGF hierarchy is specific for each TOPIC-ID and MUST


  		





  
  262
  
    
   be defined when a TOPIC-ID is assigned by the OGF Technical Director.


  		





  
  263
  
    
   These definitions MUST included information about case sensitivity,


  		





  
  264
  
    
   and in case %-escaped octets, MUST define the exact normalisation to


  		





  
  265
  
    
   use (interpret as octet, UTF-8, type of Unicode normalisation factor,


  		





  
  266
  
    
   etc.)

  269
  
    

  		





  
  270

  271
  
    
   which may allow a different set of allowed characters (URN-char) in


  		





  
  272
  
    
   the NSS than the current URN chars (adding "&" and/or "~", whilst


  		





  
  273
  
    
   removing the reserved characters "%", "/", "?" and "#").  The


  		





  
  274
  
    
   intention of this document is to only restrict the syntax of the


  		





  
  275
  
    
   TOPIC-ID, and have Grid Forum Documents specify the syntax of the


  		





  
  276
  
    

  		





  
  277

      <section title="Declaration of syntactic structure">


  		





  
  123
  
    
        


  		





  
  124
  
    
        <t>The formal syntax definitions below are given in <xref target="RFC5234">ABNF</xref>.</t>


  		





  
  125
  
    
        


  		





  
  126
  
    
        <t>The NSS in the urn:ogf names hierarchy begins with a topic


  		





  
  127
  
    
        identifier, followed by a delimiter and a topic-dependent string</t>


  		





  
  128
  
    
        


  		





  
  129
  
    
        <figure><artwork type="abnf">


  		





  
  130
  
    
   OGF-URN  =  "urn:ogf:" TOPIC-ID ":" TOPIC-SPECIFIC-STRING</artwork></figure>


  		





  
  131
  
    
        


  		





  
  132
  
    
        <t>where TOPIC-ID is a unique topic identifier for the


  		





  
  133
  
    
        specification, and TOPIC-SPECIFIC-STRING is a unique identifier 


  		





  
  134
  
    
        within the topic identifier scope.</t>


  		





  
  135
  
    
        


  		





  
  136
  
    
        <t>TOPIC-ID has the same syntax as a &lt;NID&gt; as defined in <xref target="RFC2141"/>:</t>


  		





  
  137
  
    
        


  		





  
  138
  
    
        <figure><artwork type="abnf">


  		





  
  139
  
    
   TOPIC-ID  =  ( ALPHA / DIGIT )  *31( ALPHA / DIGIT / "-" )</artwork></figure>


  		





  
  140
  
    
        


  		





  
  141
  
    
        <t>ALPHA and DIGIT are defined in <eref target="http://tools.ietf.org/html/rfc5234#appendix-B">Appendix B</eref> of <xref target="RFC5234"/>.</t>


  		





  
  142
  
    
        


  		





  
  143
  
    
        <t>The Technical Director at OGF (or it successor) assigns topic identifiers <xref target="URN-OGF"/>.</t>


  		





  
  144
  
    
        


  		





  
  145
  
    
        <t>The syntax of TOPIC-SPECIFIC-STRING is dependent on the TOPIC-ID, and SHOULD be defined by a <xref target="GFD-SERIES">Grid Forum Document</xref>. This document does not pose any additional restrictions to the TOPIC-SPECIFIC-STRING than the NSS syntax as defined by <xref target="RFC2141"/> or its successor:</t>


  		





  
  146
  
    
        


  		





  
  147
  
    
        <figure><artwork type="abnf">


  		





  
  148
  
    
   TOPIC-SPECIFIC-STRING  =  1*&lt;URN chars&gt;


  		





  
  149
  
    
</artwork></figure>


  		





  
  150
  
    
        


  		





  
  151
  
    
        <t>URN-char is defined in <eref target="http://tools.ietf.org/html/rfc2141#section-2.2">Section 2.2</eref> of <xref target="RFC2141"/>.</t>

      <section title="Rules for Lexical Equivalence">


  		





  
  192
  
    
        <t>The TOPIC-ID part of URNs in the OGF hierarchy is case insensitive. Thus, the TOPIC-ID MUST be case normalised before comparison.</t>


  		





  
  193
  
    
        


  		





  
  194
  
    
        <t>The rules for lexical equivalence of the TOPIC-SPECIFIC-STRING part of URNs in the OGF hierarchy is specific for each TOPIC-ID and MUST be defined when a TOPIC-ID is assigned by the OGF Technical Director. These definitions MUST included information about case sensitivity, and in case %-escaped octets, MUST define the exact normalisation to use (interpret as octet, UTF-8, type of Unicode normalisation factor, etc.)</t>


  		





  
  195
  
    
      </section>

      <section title="Conformance with URN Syntax">


  		





  
  198
  
    
        <t>Implementors should be aware that a revision of RFC2141 is proposed, which may allow a different set of allowed characters (URN-char) in the NSS than the current URN chars (adding "&amp;" and/or "~", whilst removing the reserved characters "%", "/", "?" and "#"). The intention of this document is to only restrict the syntax of the TOPIC-ID, and have Grid Forum Documents specify the syntax of the TOPIC-SPECIFIC-STRING. If RFC 2141 is to be updated, this document may be revised as well.</t>


  		





  
  199
  
    
        


  		





  
  200
  
    
        <t>In other words, the current definition:</t>


  		





  
  201
  
    
        <figure><artwork type="abnf">


  		





  
  202
  
    
   TOPIC-SPECIFIC-STRING  =  1*&lt;URN chars&gt;


  		





  
  203
  
    
</artwork></figure>


  		





  
  204
  
    
        


  		





  
  205
  
    
        <t>is likely to change in the future to:</t>


  		





  
  206
  
    
        <figure><artwork type="abnf">


  		





  
  207
  
    
   TOPIC-SPECIFIC-STRING  =  1*&lt;URN-char&gt;


  		





  
  208
  
    
</artwork></figure>

Box 2326<br> Joliet, Illinois 60434<br> USA<br> http://www.ogf.org/</p><p id="rfc.section.2.3.p.2">Current contact person: </p><dl class="empty"><dd>Joel Replogle<br> Email: replogle@ogf.org <span class="comment" id="Joel Replogle">[<a href="#Joel%20Replogle" class="smpl">Joel Replogle</a>: Check if a generic mail address exists]</span> </dd></dl><h2 id="rfc.section.2.4"><a href="#rfc.section.2.4">2.4</a>?Declaration of syntactic structure</h2><p id="rfc.section.2.4.p.1">The formal syntax definitions below are given in <a href="#RFC5234">ABNF</a> <cite title="Augmented BNF for Syntax Specifications: ABNF">[RFC5234]</cite>.</p><p id="rfc.section.2.4.p.2">The NSS in the urn:ogf names hierarchy begins with a topic identifier, followed by a delimiter and a topic-dependent string</p><div id="rfc.figure.u.1"></div><pre class="inline">

   OGF-URN  =  "urn:ogf:" TOPIC-ID ":" TOPIC-SPECIFIC-STRING</pre><p id="rfc.section.2.4.p.4">where TOPIC-ID is a unique topic identifier for the specification, and TOPIC-SPECIFIC-STRING is a unique identifier within the topic identifier scope.</p><p id="rfc.section.2.4.p.5">TOPIC-ID has the same syntax as a &lt;NID&gt; as defined in <a href="#RFC2141"><cite title="URN Syntax">[RFC2141]</cite></a>:</p><div id="rfc.figure.u.2"></div><pre class="inline">

This document does not pose any additional restrictions to the TOPIC-SPECIFIC-STRING than the NSS syntax as defined by <a href="#RFC2141"><cite title="URN Syntax">[RFC2141]</cite></a> or its successor:</p><div id="rfc.figure.u.3"></div><pre class="inline">

</pre><p id="rfc.section.2.4.p.11">URN-char is defined in <a href="http://tools.ietf.org/html/rfc2141#section-2.2">Section 2.2</a> of <a href="#RFC2141"><cite title="URN Syntax">[RFC2141]</cite></a>.</p><h2 id="rfc.section.2.5"><a href="#rfc.section.2.5">2.5</a>?Relevant ancillary documentation</h2><p id="rfc.section.2.5.p.1">The Technical Director at OGF (or it successor) will keep a list of assigned topic identifiers and associated documentation at <a href="http://www.ogf.org/urn/">http://www.ogf.org/urn/</a>.</p><p id="rfc.section.2.5.p.2">This website also contains information on the procedures how to register an topic identifier.</p><h2 id="rfc.section.2.6"><a href="#rfc.section.2.6">2.6</a>?Identifier uniqueness considerations</h2><p id="rfc.section.2.6.p.1">Identifier uniqueness will be enforced by the Technical Director of the Open Grid Forum.</p><h2 id="rfc.section.2.7"><a href="#rfc.section.2.7">2.7</a>?Identifier persistence considerations</h2><p id="rfc.section.2.7.p.1">The Technical Director will only assign topic identifiers for persistent resources <a href="#URN-OGF"><cite title="URN:OGF Hierarchy Registry and Documentation">[URN-OGF]</cite></a>.</p><h2 id="rfc.section.2.8"><a href="#rfc.section.2.8">2.8</a>?Process of identifier assignment</h2><p id="rfc.section.2.8.p.1">Assignment of topic identifiers is limited to the OGF and those authorities that are specifically designated by the OGF Technical Director. OGF may assign portions of its namespace (specifically, those under designated topic identifiers) for assignment by third parties.</p><p id="rfc.section.2.8.p.2">The details of this process will be specified at the OGF website <a href="#URN-OGF"><cite title="URN:OGF Hierarchy Registry and Documentation">[URN-OGF]</cite></a>.</p><p id="rfc.section.2.8.p.3">The syntax and semantics of each TOPIC-ID SHOULD be defined by a <a href="#GFD-SERIES">Grid Forum Document</a> <cite title="OGF Document Series">[GFD-SERIES]</cite> before it is assigned.</p><h2 id="rfc.section.2.9"><a href="#rfc.section.2.9">2.9</a>?Process of identifier resolution</h2><p id="rfc.section.2.9.p.1">OGF will maintain an index of all topic identifiers on its Web site, <a href="http://www.ogf.org/urn/">http://www.ogf.org/urn/</a><span class="comment" id="Joel2">[<a href="#Joel2" class="smpl">Joel2</a>: URL must be created]</span></p><p id="rfc.section.2.9.p.2">Resolution mechanism of TOPIC-SPECIFIC-STRING within each TOPIC-ID SHOULD be defined before assignment of a topic identifier.</p><h2 id="rfc.section.2.10"><a href="#rfc.section.2.10">2.10</a>?Rules for Lexical Equivalence</h2><p id="rfc.section.2.10.p.1">The TOPIC-ID part of URNs in the OGF hierarchy is case insensitive. Thus, the TOPIC-ID MUST be case normalised before comparison.</p><p id="rfc.section.2.10.p.2">The rules for lexical equivalence of the TOPIC-SPECIFIC-STRING part of URNs in the OGF hierarchy is specific for each TOPIC-ID and MUST be defined when a TOPIC-ID is assigned by the OGF Technical Director. These definitions MUST included information about case sensitivity, and in case %-escaped octets, MUST define the exact normalisation to use (interpret as octet, UTF-8, type of Unicode normalisation factor, etc.)</p><h2 id="rfc.section.2.11"><a href="#rfc.section.2.11">2.11</a>?Conformance with URN Syntax</h2><p id="rfc.section.2.11.p.1">Implementors should be aware that a revision of RFC2141 is proposed, which may allow a different set of allowed characters (URN-char) in the NSS than the current URN chars (adding "&amp;" and/or "~", whilst removing the reserved characters "%", "/", "?" and "#"). The intention of this document is to only restrict the syntax of the TOPIC-ID, and have Grid Forum Documents specify the syntax of the TOPIC-SPECIFIC-STRING.   298

  300

  261

  262
  
  
    
   be defined when a TOPIC-ID is assigned by the OGF Technical Director.


  		





  263
  
  
    
   These definitions MUST included information about case sensitivity,


  		





  264
  
  
    
   and in case %-escaped octets, MUST define the exact normalisation to


  		





  265

  266

            <t>This document lays out general definitions of and mechanisms for establishing Uniform Resource Names (URN) "namespaces".  The URN WG has defined a syntax for URNs in RFC 2141, as well as some proposed mechanisms for their resolution and use in Internet applications in RFC 3401 and RFC 3405.  The whole rests on the concept of individual "namespaces" within the URN structure.  Apart from proof-of-concept namespaces, the use of existing identifiers in URNs has been discussed in RFC 2288.  This document specifies an Internet Best Current Practices for the Internet Community, and requests discussion and suggestions for improvements.</t>
