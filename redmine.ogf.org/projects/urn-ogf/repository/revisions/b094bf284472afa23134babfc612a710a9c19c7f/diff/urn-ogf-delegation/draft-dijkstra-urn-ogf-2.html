  175
  
    

  		





  176
  
  
    
   The NSS in the urn:ogf names hierarchy begins with a topic


  		





  177
  
  
    
   identifier, followed by a delimiter and a topic-dependent string


  		





  
  176
  
    
   The NSS in the urn:ogf names hierarchy begins with a subnamespace


  		





  
  177
  
    
   identifier (SNID), followed by a delimiter and a subnamespace-


  		





  
  178
  
    
   dependent string


  		





  178
  179
  
    

  		





  179
  
  
    
      OGF-URN  =  "urn:ogf:" TOPIC-ID ":" TOPIC-SPECIFIC-STRING


  		





  
  180
  
    
      OGF-URN  =  "urn:ogf:" SNID ":" SUBNAMESPACE-SPECIFIC-STRING


  		





  180

  181
  
  
    
   where TOPIC-ID is a unique topic identifier for the specification,


  		





  182
  
  
    
   and TOPIC-SPECIFIC-STRING is a unique identifier within the topic


  		





  183
  
  
    
   identifier scope.


  		





  
  182
  
    
   where SNID is a unique subnamespace identifier for the specification,


  		





  
  183
  
    
   and SUBNAMESPACE-SPECIFIC-STRING is a unique identifier within the


  		





  
  184
  
    
   subnamespace identifier scope.


  		





  184
  185
  
    

  		





  185
  
  
    
   TOPIC-ID has the same syntax as a <NID> as defined in [RFC2141]:


  		





  
  186

  186

  187
  
  
    
      TOPIC-ID  =  ( ALPHA / DIGIT )  *31( ALPHA / DIGIT / "-" )


  		





  
  188
  
    
      SNID  =  ( ALPHA / DIGIT )  *31( ALPHA / DIGIT / "-" )


  		





  188
  189

   identifiers.

   STRING other than what is defined in the NSS syntax as defined by

  195

   and MUST be defined by a Grid Forum Document [GFD-SERIES].  This


  		





  
  197

2.10.  Rules for Lexical Equivalence


  		





  256
  256
  
    

  		





  257
  
  
    
   The TOPIC-ID part of URNs in the OGF hierarchy is case insensitive.


  		





  258
  
  
    
   Thus, the TOPIC-ID MUST be case normalised before comparison.


  		





  
  257
  
    
   The SNID part of URNs in the OGF hierarchy is case insensitive.


  		





  
  258
  
    
   Thus, the SNID MUST be case normalised before comparison.


  		





  259
  259
  
    

  		





  260
  
  
    
   The rules for lexical equivalence of the TOPIC-SPECIFIC-STRING part


  		





  261
  
  
    
   of URNs in the OGF hierarchy is specific for each TOPIC-ID and MUST


  		





  262
  
  
    
   be defined when a TOPIC-ID is assigned by the OGF Technical Director.


  		





  
  260
  
    
   The rules for lexical equivalence of the SUBNAMESPACE-SPECIFIC-STRING


  		





  
  261
  
    
   part of URNs in the OGF hierarchy is specific for each SNID and MUST


  		





  
  262
  
    
   be defined when a SNID is assigned by the OGF Technical Director.


  		





  263
  263
