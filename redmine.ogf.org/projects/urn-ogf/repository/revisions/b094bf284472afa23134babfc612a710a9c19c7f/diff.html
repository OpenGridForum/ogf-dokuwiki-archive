The OGF produces documents such as working drafts, specifications, and schemata. For more information, see <a href="http://www.ogf.org/">http://www.ogf.org/</a></p><p id="rfc.section.1.p.2">Working groups in the OGF community have expressed the need for global, distributed, persistent identifiers in working drafts and standards. Motivated by this need, the OGF would like to assign URNs to some resources in order to retain unique, permanent, location-independent names for them.</p><p id="rfc.section.1.p.3">This namespace specification is for a formal namespace.</p><h2 id="rfc.section.1.1"><a href="#rfc.section.1.1">1.1</a>?Requirements Language</h2><p id="rfc.section.1.1.p.1">The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in <a href="#RFC2119"><cite title="Key words for use in RFCs to Indicate Requirement Levels">[RFC2119]</cite></a>.</p><h1 id="rfc.section.2"><a href="#rfc.section.2">2.</a>?URN Specification for "ogf" NID</h1><h2 id="rfc.section.2.1"><a href="#rfc.section.2.1">2.1</a>?Namespace ID</h2><p id="rfc.section.2.1.p.1">"ogf" requested.</p><h2 id="rfc.section.2.2"><a href="#rfc.section.2.2">2.2</a>?Registration Information</h2><p id="rfc.section.2.2.p.1">Registration Version Number: 1<br> Registration Date: yyyy-mm-dd <span class="comment" id="RFC-Editor">[<a href="#RFC-Editor" class="smpl">RFC-Editor</a>: Please replace with the date of approval of this document for publication as an RFC]</span> </p><h2 id="rfc.section.2.3"><a href="#rfc.section.2.3">2.3</a>?Declared registrant of the namespace</h2><p id="rfc.section.2.3.p.1">Technical Director<br> Open Grid Forum<br> P.O.    OGF-URN  =  "urn:ogf:" TOPIC-ID ":" TOPIC-SPECIFIC-STRING</pre><p id="rfc.section.2.4.p.4">where TOPIC-ID is a unique topic identifier for the specification, and TOPIC-SPECIFIC-STRING is a unique identifier within the topic identifier scope.</p><p id="rfc.section.2.4.p.5">TOPIC-ID has the same syntax as a &lt;NID&gt; as defined in <a href="#RFC2141"><cite title="URN Syntax">[RFC2141]</cite></a>:</p><div id="rfc.figure.u.2"></div><pre class="inline">

OGF may assign portions of its namespace (specifically, those under designated topic identifiers) for assignment by third parties.</p><p id="rfc.section.2.8.p.2">The details of this process will be specified at the OGF website <a href="#URN-OGF"><cite title="URN:OGF Hierarchy Registry and Documentation">[URN-OGF]</cite></a>.</p><p id="rfc.section.2.8.p.3">The syntax and semantics of each TOPIC-ID MUST be defined by a <a href="#GFD-SERIES">Grid Forum Document</a> <cite title="OGF Document Series">[GFD-SERIES]</cite> before it is assigned.</p><h2 id="rfc.section.2.9"><a href="#rfc.section.2.9">2.9</a>?Process of identifier resolution</h2><p id="rfc.section.2.9.p.1">OGF will maintain an index of all topic identifiers on its Web site, <a href="http://www.ogf.org/urn/">http://www.ogf.org/urn/</a>.</p><p id="rfc.section.2.9.p.2">Resolution mechanism of TOPIC-SPECIFIC-STRING within each TOPIC-ID MUST be defined before assignment of a topic identifier.</p><h2 id="rfc.section.2.10"><a href="#rfc.section.2.10">2.10</a>?Rules for Lexical Equivalence</h2><p id="rfc.section.2.10.p.1">The TOPIC-ID part of URNs in the OGF hierarchy is case insensitive. Thus, the TOPIC-ID MUST be case normalised before comparison.</p><p id="rfc.section.2.10.p.2">The rules for lexical equivalence of the TOPIC-SPECIFIC-STRING part of URNs in the OGF hierarchy is specific for each TOPIC-ID and MUST be defined when a TOPIC-ID is assigned by the OGF Technical Director. The intention of this document is to only restrict the syntax of the TOPIC-ID, and have Grid Forum Documents specify the syntax of the TOPIC-SPECIFIC-STRING. If RFC 2141 is to be updated, this document may be revised as well.</p><p id="rfc.section.2.11.p.2">In other words, the current definition:</p><div id="rfc.figure.u.4"></div><pre class="inline">

Box 2326<br> Joliet, Illinois 60434<br> USA<br> http://www.ogf.org/<br> Email: urn@ogf.org</p><p id="rfc.section.2.3.p.2">The position of technical director is currently fulfilled by Joel Replogle.</p><h2 id="rfc.section.2.4"><a href="#rfc.section.2.4">2.4</a>?Declaration of syntactic structure</h2><p id="rfc.section.2.4.p.1">The formal syntax definitions below are given in <a href="#RFC5234">ABNF</a> <cite title="Augmented BNF for Syntax Specifications: ABNF">[RFC5234]</cite>.</p><p id="rfc.section.2.4.p.2">The NSS in the urn:ogf names hierarchy begins with a subnamespace identifier (SNID), followed by a delimiter and a subnamespace-dependent string</p><div id="rfc.figure.u.1"></div><pre class="inline">
293 
   OGF-URN  =  "urn:ogf:" SNID ":" SUBNAMESPACE-SPECIFIC-STRING</pre><p id="rfc.section.2.4.p.4">where SNID is a unique subnamespace identifier for the specification, and SUBNAMESPACE-SPECIFIC-STRING is a unique identifier within the subnamespace identifier scope.</p><p id="rfc.section.2.4.p.5">SNID has the same syntax as a &lt;NID&gt; as defined in <a href="#RFC2141"><cite title="URN Syntax">[RFC2141]</cite></a>:</p><div id="rfc.figure.u.2"></div><pre class="inline">
294 
   SNID  =  ( ALPHA / DIGIT )  *31( ALPHA / DIGIT / "-" )</pre><p id="rfc.section.2.4.p.7">ALPHA and DIGIT are defined in <a href="http://tools.ietf.org/html/rfc5234#appendix-B">Appendix B</a> of <a href="#RFC5234"><cite title="Augmented BNF for Syntax Specifications: ABNF">[RFC5234]</cite></a>.</p><p id="rfc.section.2.4.p.8">The Technical Director at OGF (or their successor) assigns subnamespace identifiers.</p><p id="rfc.section.2.4.p.9">The syntax of SUBNAMESPACE-SPECIFIC-STRING is dependent on the SNID, and MUST be defined by a <a href="#GFD-SERIES">Grid Forum Document</a> <cite title="OGF Document Series">[GFD-SERIES]</cite>. This document does not pose any additional restrictions to the SUBNAMESPACE-SPECIFIC-STRING other than what is defined in the NSS syntax as defined by <a href="#RFC2141"><cite title="URN Syntax">[RFC2141]</cite></a> or its successor:</p><div id="rfc.figure.u.3"></div><pre class="inline">
295 
   SUBNAMESPACE-SPECIFIC-STRING  =  1*&lt;URN chars&gt;
296 
</pre><p id="rfc.section.2.4.p.11">&lt;URN chars&gt; is defined in <a href="http://tools.ietf.org/html/rfc2141#section-2.2">Section 2.2</a> of <a href="#RFC2141"><cite title="URN Syntax">[RFC2141]</cite></a>.</p><h2 id="rfc.section.2.5"><a href="#rfc.section.2.5">2.5</a>?Relevant ancillary documentation</h2><p id="rfc.section.2.5.p.1">The Technical Director at OGF (or it successor) will keep a list of assigned subnamespace identifiers and associated documentation at <a href="http://www.ogf.org/urn/">http://www.ogf.org/urn/</a>  <a href="#URN-OGF"><cite title="URN:OGF Hierarchy Registry and Documentation">[URN-OGF]</cite></a>.</p><p id="rfc.section.2.5.p.2">Information on the procedures how to register an subnamespace identifier can also be found at this website.</p><h2 id="rfc.section.2.6"><a href="#rfc.section.2.6">2.6</a>?Identifier uniqueness considerations</h2><p id="rfc.section.2.6.p.1">Identifier uniqueness will be enforced by the Technical Director of the Open Grid Forum.</p><h2 id="rfc.section.2.7"><a href="#rfc.section.2.7">2.7</a>?Identifier persistence considerations</h2><p id="rfc.section.2.7.p.1">The Technical Director will only assign subnamespace identifiers for persistent resources.</p><h2 id="rfc.section.2.8"><a href="#rfc.section.2.8">2.8</a>?Process of identifier assignment</h2><p id="rfc.section.2.8.p.1">Assignment of subnamespace identifiers is limited to the OGF and those authorities that are specifically designated by the OGF Technical Director. OGF may assign portions of its namespace (specifically, those under designated subnamespace identifiers) for assignment by third parties.</p><p id="rfc.section.2.8.p.2">The details of this process will be specified at the OGF website <a href="#URN-OGF"><cite title="URN:OGF Hierarchy Registry and Documentation">[URN-OGF]</cite></a>.</p><p id="rfc.section.2.8.p.3">The syntax and semantics of each SNID MUST be defined by a <a href="#GFD-SERIES">Grid Forum Document</a> <cite title="OGF Document Series">[GFD-SERIES]</cite> before it is assigned.</p><h2 id="rfc.section.2.9"><a href="#rfc.section.2.9">2.9</a>?Process of identifier resolution</h2><p id="rfc.section.2.9.p.1">OGF will maintain an index of all subnamespace identifiers on its Web site, <a href="http://www.ogf.org/urn/">http://www.ogf.org/urn/</a>.</p><p id="rfc.section.2.9.p.2">Resolution mechanism of SUBNAMESPACE-SPECIFIC-STRING within each SNID MUST be defined before assignment of a subnamespace identifier.</p><h2 id="rfc.section.2.10"><a href="#rfc.section.2.10">2.10</a>?Rules for Lexical Equivalence</h2><p id="rfc.section.2.10.p.1">The SNID part of URNs in the OGF hierarchy is case insensitive. Thus, the SNID MUST be case normalised before comparison.</p><p id="rfc.section.2.10.p.2">The rules for lexical equivalence of the SUBNAMESPACE-SPECIFIC-STRING part of URNs in the OGF hierarchy is specific for each SNID and MUST be defined when a SNID is assigned by the OGF Technical Director. These definitions MUST included information about case sensitivity, and in case %-escaped octets, MUST define the exact normalisation to use (e.g. interpret as octet, interpret as UTF-8, specify type of Unicode normalisation factor, etc.)</p><h2 id="rfc.section.2.11"><a href="#rfc.section.2.11">2.11</a>?Conformance with URN Syntax</h2><p id="rfc.section.2.11.p.1">At the moment of writing, a revision of RFC2141 is proposed, which may allow a different set of allowed characters (URN-char) in the NSS than the current URN chars (adding "&amp;" and/or "~", whilst removing the reserved characters "%", "/", "?" and "#"). The intention of this document is to only restrict the syntax of the SNID, and have Grid Forum Documents specify the syntax of the SUBNAMESPACE-SPECIFIC-STRING. If RFC 2141 is to be updated, this document may be revised as well.</p><p id="rfc.section.2.11.p.2">In other words, the current definition:</p><div id="rfc.figure.u.4"></div><pre class="inline">
297 
   SUBNAMESPACE-SPECIFIC-STRING  =  1*&lt;URN chars&gt;
298 298 
</pre><p id="rfc.section.2.11.p.4">is likely to change in the future to:</p><div id="rfc.figure.u.5"></div><pre class="inline">
299 
   TOPIC-SPECIFIC-STRING  =  1*&lt;URN-char&gt;
300 
</pre><p id="rfc.section.2.11.p.6">It is RECOMMENDED that OGF URN strings be chosen in characters acceptable by both RFC2141 and the expected revision.</p><h2 id="rfc.section.2.12"><a href="#rfc.section.2.12">2.12</a>?Validation mechanism</h2><p id="rfc.section.2.12.p.1">The validation mechanism of URNs in the OGF hierarchy is specific for each TOPIC-ID and MUST be defined when a TOPIC-ID is assigned by the OGF Technical Director.</p><p id="rfc.section.2.12.p.2">URNs in the OGF hierarchy without an assigned TOPIC-ID are considered to be invalid.</p><h2 id="rfc.section.2.13"><a href="#rfc.section.2.13">2.13</a>?Scope</h2><p id="rfc.section.2.13.p.1">Global URNs, relevant for the distributed computing community in general, and the Open Grid Forum in particular.</p><h1 id="rfc.section.3"><a href="#rfc.section.3">3.</a>?Examples (Informative)</h1><p id="rfc.section.3.p.1">Since no topic identifiers have been defined yet, no actual examples can be given.
