The core of the OCCI model is simple. The central resource type is


  		





  123
  
  
    
Resource and contains a number of common attributes that


  		





  124
  
  
    
domain-specific Resource types inherit. These Resource types are


  		





  125
  
  
    
complemented by two additional types, Link and Action. Accompanying


  		





  126
  
  
    
these, Kind and Category provide the fabric for a safe extension model


  		





  127
  
  
    
towards domain-specific usage. For compliance with OCCI Core, all


  		





  128
  
  
    
types MUST be implemented. The UML class diagram \ref{fig:occi_core}


  		





  129
  
  
    
gives an overview of the model.


  		





  
  130
  
    
The heart of the OCCI model is the \hl{Resource} type. To expose a resource


  		





  
  131
  
    
through the OCCI API a domain-specific sub-type of \hl{Resource} is used to


  		





  
  132
  
    
represent the resource in the OCCI model.


  		





  
  133
  
    
A resource can be basically anything, e.g. a virtual machine, a job in a job


  		





  
  134
  
    
submission system, a user, etc.


  		





  
  135
  
    
%


  		





  
  136
  
    
The \hl{Resource} type contains a number of common attributes that


  		





  
  137
  
    
domain-specific \hl{Resource} types inherit. The \hl{Resource} type is


  		





  
  138
  
    
complemented by the \hl{Link} type which associates one \hl{Resource} instance


  		





  
  139
  
    
with another.


  		





  
  140
  
    
%


  		





  
  141
  
    
The \hl{Link} type also contain a number of common attributes that


  		





  
  142
  
    
domain-specific \hl{Link} types inherit.


  		





  
  143
  
    

  		





  
  144
  
    
\hl{Kind} is an abstract type which both \hl{Resource} and \hl{Link} inherit.


  		





  
  145
  
    
Each type inheriting \hl{Kind} is identified by a unique \hl{Type} instance. A


  		





  
  146
  
    
\hl{Type} define the capabilities of a \hl{Kind} in terms of additional


  		





  
  147
  
    
attributes and \hl{Action} types.


  		





  
  148
  
    
%


  		





  
  149
  
    
The \hl{Type} type comprise the identification and classification system built


  		





  
  150
  
    
into the OCCI model. \hl{Type} is a specialisation of \hl{Category} and


  		





  
  151
  
    
introduce additional capabilities in terms of \hl{Action} types.
