\subsection{Linking Infrastructure Resources}


  		





  366
  
  
    
In order to create entities like virtual data centres or virtual clusters, it is necessary to allow the linking of the various infrastructure Resource types. This can be accomplished extending the core specification Link entity as the Link entity does not include enough information about specific types of infrastructure links.


  		





  
  366
  
    
In order to create entities like virtual data centres or virtual clusters, it is necessary to allow the linkage of the previously defined infrastructure \hl{Resource} sub-types. This is accomplished by extending (sub-classing) the OCCI Core Model \hl{Link} base type. This is done as the \hl{Link} base type cannot fully represent specific types of infrastructure links (e.g. links to storage or networks).

\subsubsection{Linking to Network}


  		





  
  369
  
    
\hl{NetworkInterface} represents an L2 client device (e.g. network adapter). It can be extended using the mix-in mechanism or sub-classed to support L3/L4 capabilities such as TCP/IP etc. 


  		





  370
  370
  
    

  		





  371
  
  
    
The Networkinterface Link type is assigned the http://schemas.ogf.org/occi/infrastructure\#networkinterface Category. A Networkinterface Link instance MUST use and expose this Category.


  		





  
  371
  
    
The \hl{NetworkInterface} type is assigned the \textit{http://schemas.ogf.org/occi/infrastructure\#networkinterface} \hl{Kind}. A \hl{NetworkInterface} instance MUST use and expose this \hl{Kind}. A \hl{NetworkInterface} MUST be related to the \textit{http://schemas.ogf.org/occi/core\#link} \hl{Kind}.
