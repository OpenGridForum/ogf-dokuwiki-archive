To adapt traffic to and from different layers, an \emph{adaptation} relation is used. An adaptation consists of two unidirectional components, an \emph{adaptation sink} to go from a server layer port to a client layer port, and an \emph{adaptation source} for the other direction.

To adapt traffic to and from different layers, an \emph{adaptation} relation is used. An adaptation consists of two unidirectional components, an \emph{adaptationSink} to go from a server layer port to a client layer port, and an \emph{adaptationSource} for the other direction.

A \emph{Link} object describes that there is a unidirectional connection from one \emph{Port} to another. These ports are identified using the \emph{source} and \emph{sink} relationships.


  		





  110
  111
  
    

  		





  111
  
  
    
A \emph{Link} MUST have an attribute \emph{type} which is either \emph{Link} or \emph{Crossconnect}. When the type is Crossconnect, the source and sink MUST be part of the same node.


  		





  
  112
  
    
A \emph{Link} \MUST{} have an attribute \emph{type} which is either \emph{Link} or \emph{Crossconnect}. When the type is Crossconnect, the source and sink \MUST{} be part of the same node.


  		





  112
  113
  
    

  		





  113
  
  
    
A \emph{Link} should have a \emph{capacity}  attribute which describes the capacity of the link in bytes per second. This value should correspond to the actual amount of data that can be transported over the link, excluding overhead.


  		





  
  114
  
    
A \emph{Link} \SHOULD{} have a \emph{capacity}  attribute which describes the capacity of the link in bytes per second. This value \SHOULD{} correspond to the actual amount of data that can be transported over the link, excluding overhead.
