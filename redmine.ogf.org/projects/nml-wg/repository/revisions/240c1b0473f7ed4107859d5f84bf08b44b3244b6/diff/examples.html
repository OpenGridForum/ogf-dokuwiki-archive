\subsubsection{Links, Segments and Paths}


  		





  
  142
  
    

  		





  
  143
  
    
A \emph{Link} object can refer to any link connection. A link segment and an end-to-end path are both described by a \emph{Link} object. This is by design, since it is easy to extend a \emph{Link}, or to describe a partition of a \emph{Link}.

Note that in this example Port \texttt{port\_Y:out} is the source of both \textsc{linkB:YZ} and of \textsc{linkBC:YW}. If a single topology description would contain the full link and the partitioning, a path finding algoritm \MUST{} be aware that the fact that Port \textsc{port\_Y:out} is the source of two NML Links, does not mean it multicast to different network links. For this reasons, it is \RECOMMENDED{} that applications either add metadata about the type of link, or specify that in certain messages, only one particular type of Link \MUST{} be used.
