\emph{Relations} describe how different network objects can be combined to form a network topology description. The relations have been described above, but for ease of reference we also give a full list and definition here (in alphabetical order):


  		





  
  171
  
    

  		





  
  172
  
    
\begin{description}


  		





  
  173
  
    
    \item[adaptation] is the bi-directional equivalent of the source and sink adaptation combination.    


  		





  
  174
  
    
    \item[adaptation sink] goes from a server layer port to a client layer port, describing the way that data is passed between these two layers.


  		





  
  175
  
    
    \item[adaptation source]  is the reverse of the sink adaptation, i.e. it goes from a client layer port to a server layer port.


  		





  
  176
  
    
    \item[has port] describes the relation between a node and a port.


  		





  
  177
  
    
    \item[has service] describes the relation between a node and a service that it provides.


  		





  
  178
  
    
    \item[implemented by] is a relation from a node to a node, describing that the source node is a virtualized node on the sink node of the relation.


  		





  
  179
  
    
    \item[located at] is a relation between a network object and a location object.


  		





  
  180
  
    
    \item[sink] is the connection between the end of the link and the sink port.


  		





  
  181
  
    
    \item[source] describes the connection of a port to the source port of the link.


  		





  
  182
  
    
\end{description}
