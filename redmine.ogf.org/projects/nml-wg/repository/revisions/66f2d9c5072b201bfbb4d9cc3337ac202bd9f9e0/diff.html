\subsection{Scope}


  		





  
  33
  
    

  		





  
  34
  
    
The Network Markup Language is specifically designed to describe connection-oriented 


  		





  
  35
  
    
network topologies. It is specially aimed at logical network description, multi-layer 


  		





  
  36
  
    
network descriptions (including virtualised networks) and multi-domain network 


  		





  
  37
  
    
descriptions. It can not only describe a statics topology, but also its capabilities 


  		





  
  38
  
    
and its configuration.


  		





  
  39
  
    

  		





  
  40
  
    
NML can also be used to describe physical networks or packet-oriented networks, 


  		





  
  41
  
    
although the current base schema current version does contain classes or properties 


  		





  
  42
  
    
to explicitly deal with signal degradation, or complex routing tables.


  		





  
  43
  
    

  		





  
  44
  
    
NML only attempts to describe the data plane of a computer network, not the control 


  		





  
  45
  
    
plane. It does contain extension mechanism to easily tie it with network provisioning 


  		





  
  46
  
    
standards, and with network monitoring standards.


  		





  
  47
  
    

  		





  
  48
  
    
Finally, you will not find a definition for the terms \emph{Network} or \emph{capacity} in this 


  		





  
  49
  
    
document. This has been a conscious choice. The term \emph{Network} has become 


  		





  
  50
  
    
so widely used for so many diverse meanings that it is impossible to create a 


  		





  
  51
  
    
definition that everyone can agree on, while still expressing something useful.


  		





  
  52
  
    
The term \emph{capacity} is used by different technologies in such a different 


  		





  
  53
  
    
way (e.g. including or excluding the packet overhead) that we think it is better 


  		





  
  54
  
    
to let technology-specific extensions make an explicit definition.
