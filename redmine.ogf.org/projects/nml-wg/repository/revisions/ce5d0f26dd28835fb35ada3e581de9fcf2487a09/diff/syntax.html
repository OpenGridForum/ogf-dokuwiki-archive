The Network Markup Language has two different normative syntaxes. The syntaxes are in regular XML defined using an XML Schema (XSD), and another in OWL RDF/XML syntax, defined in an OWL schema.


  		





  6
  7
  
    
The OWL syntax is aimed at Semantic Web-oriented applications, the XML syntax is suitable for any application. These syntaxes are defined in Appendices \ref{s:xmlschema} and \ref{s:owlschema} respectively.

  14
  
    
\begin{lstlisting}

  16
  
    
\end{lstlisting}

Explicit relation are represented as a \texttt{<nml:Relation>} XML element, with the domain as the parent element, and the range as the child elment. Implicit relations are not given: the range object is represented as a XML child element of the doamin. For example:

  47
  
    
<nml:Port id="urn:ogf:network:example.net:2012:port_X:out">


  		





  
  48
  
    
  <nml:Relation type="http://schemas.ogf.org/nml/2012/10/base#isSource">


  		





  
  49

  51
  
    
</nml:Port>

\subsection{OWL RDF/XML Syntax} % (fold)


  		





  
  65
  
    
\label{sub:owl}


  		





  
  66
  
    

  		





  
  67
  
    
A NML object is represented as a RDF subject.


  		





  
  68
  
    

  		





  
  69
  
    
A NML attribute or parameter is represented as a predicate. 


  		





  
  70
  
    

  		





  
  71
  
    
For example:


  		





  
  72
  
    
\begin{lstlisting}


  		





  
  73
  
    
<nml:Port rdf:about="urn:ogf:network:example.net:2012:port_X.1501:in">


  		





  
  74
  
    
  <nml:name>VLAN 1501 at Port X (in)</nml:name>


  		





  
  75
  
    
  <nml:hasLabeL rdf:resource="#e43a63"/>


  		





  
  76
  
    
</nml:Port>


  		





  
  77
  
    
<nml:Label rdf:about="#e43a63">


  		





  
  78
  
    
  <labeltype="http://schemas.ogf.org/nml/2012/10/ethernet#vlan">1501</nml:Label>


  		





  
  79
  
    
</nml:Label>


  		





  
  80
  
    
\end{lstlisting}


  		





  
  81
  
    

  		





  
  82
  
    
Relations are represented as a RDF triplet, with the full URI of the attribute or parameter.  For example:


  		





  
  83
  
    
\begin{lstlisting}


  		





  
  84
  
    
<nml:Port rdf:about="urn:ogf:network:example.net:2012:port_X:out">


  		





  
  85
  
    
  <nml:isSource rdf:resource=" urn:ogf:network:example.net:2012:linkA:XY"/>


  		





  
  86
  
    
</nml:Port>


  		





  
  87
  
    
\end{lstlisting}
