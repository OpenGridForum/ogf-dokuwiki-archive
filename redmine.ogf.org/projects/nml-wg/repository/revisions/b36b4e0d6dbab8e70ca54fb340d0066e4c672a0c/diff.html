\subsubsection{Ordered Lists} % (fold)


  		





  961
  982
  
    
\label{ssub:ordered_lists}


  		





  962
  983
  
    

  		





  963
  
  
    
The XML and OWL schemas have a different approach to defining the NML \emph{List}. In XML a \emph{List} can be constructed using an additional \emph{next} attribute on an object. The OWL syntax does not use attributes, and using a\emph{next} object property on an object would cause confusion if an item is in multiple lists. Therefore we have introduced a \emph{ListItem} object, which holds the value through the \emph{item} relationship, and also has an optional \emph{next} item to relate to the next item of the list. See also the SerialCompoundLink examples in the example section.


  		





  
  984
  
    
The range of a \emph{isSerialCompoundLink} is an \emph{OrderedList}.


  		





  
  985
  
    

  		





  
  986
  
    
Neither XML nor OWL uses the  \emph{OrderedList} directly in the syntax, and have a different way of constructing ordered lists. XML uses the \emph{next} relation, while OWL uses the \emph{ListItem} class, and the \emph{item} and \emph{next} relations.


  		





  
  987
  
    

  		





  
  988
  
    
A \emph{OrderedList} and \emph{ListItem} behave as classes, while \emph{item} and \emph{next} behave like relations, with the exception that these classes and relation are local in scope.


  		





  
  989
  
    

  		





  
  990
  
    
This means that these relations are only valid within the scope of a given \emph{OrderedList}, but may not be valid in scope of a different \emph{OrderedList}. It also means that any identifier given to these classes may change when the objects are codified in a syntax.


  		





  
  991
  
    

  		





  
  992
  
    
For example, consider the following two decompositions of a \emph{Link} into shorter \emph{Links}:


  		





  
  993
  
    

  		





  
  994
  
    
\begin{itemize}


  		





  
  995
  
    
    \item \nmlrelation{Link\_1\_2\_3}{\vrule width0pt height1.5ex}{isSerialCompoundLink}{}{\lower3.5ex\vbox{\hbox{\framebox{Link\_1}}\hbox{\framebox{Link\_2}}\hbox{\framebox{Link\_3}}}}


  		





  
  996
  
    
    \item \nmlrelation{Link\_1\_2\_3}{\vrule width0pt height1.5ex}{isSerialCompoundLink}{}{\lower2ex\vbox{\hbox{\framebox{Link\_1}}\hbox{\framebox{Link\_2\_3}}}}


  		





  
  997
  
    
\end{itemize}


  		





  
  998
  
    

  		





  
  999
  
    
Observe that in the first OrderedList, there is a next relation from \texttt{Link\_1} to \texttt{Link\_2}, while in the second OrderedList, the next relation is from \texttt{Link\_1} to \texttt{Link\_2\_3}.


  		





  
  1000
  
    

  		





  
  1001
  
    
In XML a \emph{Ordered List} can be constructed by pointing to the first object in the list, and using a \emph{next} relation between consecutive objects in the list.


  		





  
  1002
  
    

  		





  
  1003
  
    
In OWL a \emph{Ordered List} can be constructed by creating as many \emph{ListItem} object as there are items in the list, and correlating each \emph{ListItem} object with the item in the List using the \emph{item} relation. The Ordered List is than constructed by pointing to the first \emph{ListItem} in the list, and using a \emph{next} relation between consecutive \emph{ListItem}s in the list.
