Larger networks may want to publish details of their network topology as a whole, as well as detaild about regional segments, without publishing details of the actual devices. NML allows the publication of a hierarchical Topology tree, where the top-level Topology has a \emph{hasTopology} relation with smaller Topologies. These smaller Topologies must be fully enclosed -- the \emph{hasTopology} relation can not be used to relate partial overlapping Topologies.

However, this is not entirely correct: in the above description data coming from \texttt{Port\_X:in} would also be forwarded to \texttt{Port\_X:out}. However, the Ethernet technology prevents data returning on the same interface.

NML introduced the \emph{noReturnTraffic} parameter to deal with this situation: if the \emph{noReturnTraffic} parameter of a Link is true, there is no data transport from a source to a sink if the source and sink are grouped together in a BidirectionalPort group.

\nmlrelation{Link VLAN\_42}{}{noReturnTraffic}{}{"\texttt{true}"} \\
