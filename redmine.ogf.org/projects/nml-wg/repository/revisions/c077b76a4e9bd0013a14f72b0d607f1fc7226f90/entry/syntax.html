\section{Syntax}


    		
  

  

  

    
      3
    
    
      
\label{s:syntax}


    		
  

  

  

    
      4
    
    
      

    		
  

  

  

    
      5
    
    
      
The Network Markup Language has two different normative syntaxes. The syntaxes are in regular XML defined using an XML Schema (XSD), and another in OWL RDF/XML syntax, defined in an OWL schema.


    		
  

  

  

    
      6
    
    
      
The OWL syntax is aimed at Semantic Web-oriented applications, the XML syntax is suitable for any application. These syntaxes are defined in Appendices \ref{s:xmlschema} and \ref{s:owlschema} respectively.

\subsection{Combining Object Descriptions} % (fold)


    		
  

  

  

    
      9
    
    
      
\label{sub:combining_descriptions}


    		
  

  

  

    
      10
    
    
      

    		
  

  

  

    
      11
    
    
      
A given object may have multiple attributes and relations. These attributes 


    		
  

  

  

    
      12
    
    
      
and relations may be described in different places in a syntax. It is up to 


    		
  

  

  

    
      13
    
    
      
the parser to combine all attributes and relations.


    		
  

  

  

    
      14
    
    
      

    		
  

  

  

    
      15
    
    
      
NML currently does not have a mechanism to check if a given description of an 


    		
  

  

  

    
      16
    
    
      
object is \emph{complete}. Thus, it does not distinguish between a full description 


    		
  

  

  

    
      17
    
    
      
of an object or merely a pointer to an object.


    		
  

  

  

    
      18
    
    
      

    		
  

  

  

    
      19
    
    
      
% Such distinction may be introduced by a future version of NML.


    		
  

  

  

    
      20
    
    
      

    		
  

  

  

    
      21
    
    
      
Parsers should be aware that the NML descriptions do not provide any guarantee 


    		
  

  

  

    
      22
    
    
      
regarding the integrity nor the authenticity of the description. Parsers are 


    		
  

  

  

    
      23
    
    
      
advised to use external mechanism to avoid that an erroneous description of an 


    		
  

  

  

    
      24
    
    
      
object in one (possibly malicious) topology description pollutes a correct 


    		
  

  

  

    
      25
    
    
      
description of the same object in another topology description.


    		
  

  

  

    
      26
    
    
      

    		
  

  

  

    
      27

\subsection{Ordered Lists} % (fold)


    		
  

  

  

    
      30
    
    
      
\label{sub:ordered_lists}


    		
  

  

  

    
      31
    
    
      

    		
  

  

  

    
      32
    
    
      
The range of a \emph{isSerialCompoundLink} is an \emph{OrderedList}.


    		
  

  

  

    
      33
    
    
      

    		
  

  

  

    
      34
    
    
      
Neither XML nor OWL uses the  \emph{OrderedList} directly in the syntax, and have a different way of constructing ordered lists. XML uses the \emph{next} relation, while OWL uses the \emph{ListItem} class, and the \emph{item} and \emph{next} relations.


    		
  

  

  

    
      35
    
    
      

    		
  

  

  

    
      36
    
    
      
A \emph{OrderedList} and \emph{ListItem} behave as classes, while \emph{item} and \emph{next} behave like relations, with the exception that these classes and relation are local in scope.


    		
  

  

  

    
      37
    
    
      

    		
  

  

  

    
      38
    
    
      
This means that these relations are only valid within the scope of a given \emph{OrderedList}, but may not be valid in scope of a different \emph{OrderedList}. It also means that any identifier given to these classes may change when the objects are codified in a syntax.


    		
  

  

  

    
      39
    
    
      

    		
  

  

  

    
      40
    
    
      
For example, consider the following two decompositions of \emph{Link} \texttt{Link\_1\_2\_3} into shorter \emph{Links}:


    		
  

  

  

    
      41
    
    
      

    		
  

  

  

    
      42
    
    
      
\begin{itemize}


    		
  

  

  

    
      43
    
    
      
    \item \nmlrelation{Link\_1\_2\_3}{\vrule width0pt height1.5ex}{isSerialCompoundLink}{}{\lower3.5ex\vbox{\hbox{\framebox{Link\_1}}\hbox{\framebox{Link\_2}}\hbox{\framebox{Link\_3}}}}


    		
  

  

  

    
      44
    
    
      
    \item \nmlrelation{Link\_1\_2\_3}{\vrule width0pt height1.5ex}{isSerialCompoundLink}{}{\lower2ex\vbox{\hbox{\framebox{Link\_1}}\hbox{\framebox{Link\_2\_3}}}}


    		
  

  

  

    
      45
    
    
      
\end{itemize}


    		
  

  

  

    
      46
    
    
      

    		
  

  

  

    
      47
    
    
      
In the first OrderedList, there is a \emph{next} relation from \texttt{Link\_1} to \texttt{Link\_2}, while in the second OrderedList, the \emph{next} relation is from \texttt{Link\_1} to \texttt{Link\_2\_3}.


    		
  

  

  

    
      48
    
    
      

    		
  

  

  

    
      49
    
    
      
In XML a \emph{Ordered List} can be constructed by pointing to the first object in the list, and using a \emph{next} relation between consecutive objects in the list.


    		
  

  

  

    
      50
    
    
      

    		
  

  

  

    
      51
    
    
      
In OWL a \emph{Ordered List} can be constructed by creating as many \emph{ListItem} object as there are items in the list, and correlating each \emph{ListItem} object with the item in the List using the \emph{item} relation. The Ordered List is than constructed by pointing to the first \emph{ListItem} in the list, and using a \emph{next} relation between consecutive \emph{ListItem}s in the list.


    		
  

  

  

    
      52
    
    
      

    		
  

  

  

    
      53
    
    
      
See also the isSerialCompoundLink examples in the example section.


    		
  

  

  

    
      54
