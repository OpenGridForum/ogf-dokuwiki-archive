\section{Object Identifiers}


  		





  ......




  12
  14
  
    

  		





  13
  15
  
    
Implementations that receive a network topology description MUST be prepared to accept any valid URI as an identifier.


  		





  14
  16
  
    

  		





  15
  
  
    
Implementations that publish a network topology description instance identifiers MAY adhere to the syntax of Global Network Identifiers as defined in \ref{gfd-urn-ogf-network}, which ensures global uniqueness and that easy recognition of Network Object instances.


  		





  
  17
  
    
Implementations that publish a network topology description instance identifiers MAY adhere to the syntax of Global Network Identifiers as defined in \cite{gfd-urn-ogf-network}, which ensures global uniqueness and that easy recognition of Network Object instances.


  		





  16
  18
  
    

  		





  17
  19
  
    
Two different Network Objects instance MUST have two different identifiers.


  		





  18
  20
  
    

  		





  ......




  31
  33
  
    
For example the following identifiers are equivalent:


  		





  32
  34
  
    

  		





  33
  35
  
    
\begin{quote}


  		





  34
  
  
    
  \texttt{1 - urn:ogf:network:example.net:2012:local\_string\_1234}


  		





  
  36
  
    
  \texttt{1 - urn:ogf:network:example.net:2012:local\_string\_1234}\\


  		





  35
  37
  
    
  \texttt{2 - URN:OGF:network:EXAMPLE.NET:2012:Local\_String\_1234}


  		





  36
  38
  
    
\end{quote}


  		





  37
  39
  
    

  		





  38
  40
  
    
while the following identifiers are not equivalent (in this case, the percentage encoding even make URI \#3 an invalid Global Network Identifier.):


  		





  39
  41
  
    

  		





  40
  42
  
    
\begin{quote}


  		





  41
  
  
    
  \texttt{1 - urn:ogf:network:example.net:2012:local\_string\_1234}


  		





  
  43
  
    
  \texttt{1 - urn:ogf:network:example.net:2012:local\_string\_1234}\\


  		





  42
  44
  
    
  \texttt{3 - urn:ogf:network:example.net:2012:local\%5Fstring\%5F1234}


  		





  43
  45
  
    
\end{quote}


  		





  44
  46
  
    

  		





  ......




  48
  50
  
    

  		





  49
  51
  
    
Parsers MUST be prepared to accept identifiers of up to 255 characters in length.


  		





  50
  52
  
    

  		





  51
  
  
    
A Parser SHOULD verify if an identifier adheres to the general URI syntax rules, as specified in RFC 3986.


  		





  
  53
  
    
A Parser SHOULD verify if an identifier adheres to the general URI syntax rules, as specified in RFC 3986~\cite{rfc3986}.


  		





  52
  54
  
    

  		





  53
  55
  
    
Parsers SHOULD reject identifiers which do not adhere to the specified rules. A parser encountering an invalid identifier SHOULD reply with an error code that includes the malformed identifier, but MAY accept the rest of the message, after purging all references to the Network Object with the malformed identifier.
